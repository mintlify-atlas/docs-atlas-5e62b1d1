---
title: Chrome storage usage
description: How Click-n-Color stores colors, settings, themes, and state using chrome.storage.local
---

Click-n-Color uses `chrome.storage.local` to persist user data. All data is stored locally on the user's device and never synced or transmitted externally.

## Storage structure

The extension stores four main data keys:

```javascript
{
  savedColors: string[],      // Array of color strings
  lastPickedColor: string,     // Most recently picked color
  settings: object,            // User preferences
  theme: string                // Selected UI theme
}
```

## Storage keys

### savedColors

**Type**: `string[]`

**Purpose**: Stores the user's color palette

**Format**: Array of RGB color strings

**Example**:
```json
[
  "rgb(138, 43, 226)",
  "rgb(0, 191, 255)",
  "rgb(255, 20, 20)"
]
```

**Initialization**:

```javascript background.js:5-7
if (!Array.isArray(result.savedColors)) {
    initialState.savedColors = [];
}
```

**Maximum size**: Configurable via `settings.maxPaletteSize` (default: 50 colors)

<Info>
Colors are stored in RGB format internally, even though the UI can display them in HEX or HSL formats.
</Info>

### Adding colors to palette

When a color is picked, it's added to the beginning of the array:

```javascript background.js:102-115
chrome.storage.local.get(['savedColors', 'settings'], (result) => {
    let savedColors = Array.isArray(result.savedColors) ? result.savedColors : [];
    const settings = result.settings && typeof result.settings === 'object' ? result.settings : {};
    const parsedMax = parseInt(settings.maxPaletteSize, 10);
    const maxPaletteSize = Number.isFinite(parsedMax)
        ? Math.min(100, Math.max(10, parsedMax))
        : 50;

    if (!savedColors.includes(message.color)) {
        savedColors.unshift(message.color);
        if (savedColors.length > maxPaletteSize) {
            savedColors = savedColors.slice(0, maxPaletteSize);
        }
    }
    // ...
});
```

<Note>
Duplicate colors are not added to the palette - each color appears only once.
</Note>

### lastPickedColor

**Type**: `string | null`

**Purpose**: Tracks the most recently picked color for display in the popup

**Format**: RGB color string or `null` if no color has been picked

**Example**: `"rgb(138, 43, 226)"`

**Initialization**:

```javascript background.js:9-11
if (typeof result.lastPickedColor === 'undefined') {
    initialState.lastPickedColor = null;
}
```

**Updated when**:
- A color is successfully picked
- Set to the color value, even if it's already in the palette

```javascript background.js:117-119
chrome.storage.local.set({
    savedColors,
    lastPickedColor: message.color
});
```

<Info>
`lastPickedColor` is used to restore the color preview when reopening the popup.
</Info>

### settings

**Type**: `object`

**Purpose**: Stores user preferences and configuration

**Structure**:

```javascript popup.js:22-26
{
    defaultFormat: 'rgb',     // 'rgb' | 'hex' | 'hsl'
    maxPaletteSize: 50,       // 10-100
    autoCopy: true            // boolean
}
```

**Default values**:

| Setting | Default | Constraints |
|---------|---------|-------------|
| `defaultFormat` | `'rgb'` | Must be `'rgb'`, `'hex'`, or `'hsl'` |
| `maxPaletteSize` | `50` | Integer between 10 and 100 |
| `autoCopy` | `true` | Boolean |

**Validation**:

```javascript popup.js:39-58
function sanitizeSettings(rawSettings) {
    const safeSettings = rawSettings && typeof rawSettings === 'object' ? rawSettings : {};
    const defaultFormat = ALLOWED_FORMATS.includes(safeSettings.defaultFormat)
        ? safeSettings.defaultFormat
        : 'rgb';

    const parsedPaletteSize = parseInt(safeSettings.maxPaletteSize, 10);
    const maxPaletteSize = Number.isFinite(parsedPaletteSize)
        ? Math.min(100, Math.max(10, parsedPaletteSize))
        : 50;

    const autoCopy = typeof safeSettings.autoCopy === 'boolean'
        ? safeSettings.autoCopy
        : true;

    return {
        defaultFormat,
        maxPaletteSize,
        autoCopy
    };
}
```

<Warning>
Settings are validated on load to prevent invalid values from breaking the extension.
</Warning>

### theme

**Type**: `string`

**Purpose**: Stores the user's selected UI theme

**Allowed values**: `'default'`, `'light'`, `'blue'`

**Default**: `'default'`

**Validation**:

```javascript popup.js:35-37
function sanitizeTheme(themeValue) {
    return ALLOWED_THEMES.includes(themeValue) ? themeValue : 'default';
}
```

**Theme application**:

```javascript popup.js:543-566
function applyTheme(theme) {
    const root = document.documentElement;
    // Reset to default theme
    root.style.setProperty('--primary', '#8a2be2');
    root.style.setProperty('--secondary', '#00bfff');

        root.style.setProperty('--background', '#f5f5f5');

        root.style.setProperty('--hover', '#d0d0d0');

        root.style.setProperty('--secondary', '#64b5f6');
        root.style.setProperty('--accent', '#ff6d00');
        root.style.setProperty('--background', '#0a1929');
        root.style.setProperty('--card', '#102a43');
    }
}
```

<Info>
Themes use CSS custom properties to dynamically change the UI appearance.
</Info>

## Storage initialization

On first installation, the service worker initializes empty storage:

```javascript background.js:1-19
chrome.runtime.onInstalled.addListener(() => {
    chrome.storage.local.get(['savedColors', 'lastPickedColor'], (result) => {
        const initialState = {};

        if (!Array.isArray(result.savedColors)) {
            initialState.savedColors = [];
        }

        if (typeof result.lastPickedColor === 'undefined') {
            initialState.lastPickedColor = null;
        }

        if (Object.keys(initialState).length > 0) {
            chrome.storage.local.set(initialState, () => {
                console.log("The Color Picker extension initialized");
            });
        }
    });
});
```

<Note>
The initialization only sets missing keys, preserving existing data during updates.
</Note>

## Storage change listeners

The popup listens for storage changes to update the UI in real-time:

```javascript popup.js:140-153
chrome.storage.onChanged.addListener((changes, areaName) => {
    if (areaName !== 'local') {
        return;
    }

    if (changes.lastPickedColor && isUsableColor(changes.lastPickedColor.newValue)) {
        setSelectedColor(changes.lastPickedColor.newValue);
    }

    if (changes.savedColors) {
        const updatedColors = Array.isArray(changes.savedColors.newValue) ? changes.savedColors.newValue : [];
        renderSavedColors(updatedColors);
    }
});
```

This enables:
- Automatic UI updates when colors are picked
- Synchronization between service worker and popup
- Instant palette refresh

## Storage limits

### chrome.storage.local limits

- **Maximum total storage**: 10 MB (Chrome default)
- **Unlimited items**: No per-item limit

### Extension-enforced limits

- **Maximum palette size**: 100 colors (enforced in code)
- **Minimum palette size**: 10 colors
- **Default palette size**: 50 colors

```javascript background.js:105-108
const maxPaletteSize = Number.isFinite(parsedMax)
    ? Math.min(100, Math.max(10, parsedMax))
    : 50;
```

<Warning>
When the palette exceeds `maxPaletteSize`, the oldest colors are automatically removed.
</Warning>

## Storage persistence

`chrome.storage.local` data is:

- **Persistent**: Survives browser restarts
- **Local-only**: Never synced to Chrome account or cloud
- **Extension-scoped**: Isolated from other extensions and websites
- **Cleared on uninstall**: All data removed when extension is uninstalled

<Info>
Users can manually clear extension storage via Chrome's extension settings.
</Info>

## Data export

Users can export their color palette without accessing storage directly:

**JSON export**:
```javascript popup.js:375-389
function exportAsJson() {
    chrome.storage.local.get(['savedColors'], (result) => {
        const savedColors = result.savedColors || [];
        const dataStr = JSON.stringify(savedColors, null, 2);
        const dataUri = 'data:application/json;charset=utf-8,' + encodeURIComponent(dataStr);
        
        const exportFileName = 'color-palette.json';
        const linkElement = document.createElement('a');
        linkElement.setAttribute('href', dataUri);
        linkElement.setAttribute('download', exportFileName);
        linkElement.click();
    });
}
```

This exports `savedColors` as a downloadable JSON file for backup or sharing.
